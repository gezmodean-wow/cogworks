# Cogworks feedback

Library-level issues tracked here. Per-cog issues live in each cog's own `feedback/` directory (see `~/.chronoforge/CONVENTIONS.md`).

## Directories

- `collection/` — still gathering info, not yet scoped to a release
- `releases/v<X.Y.Z>/` — scoped to a target library minor-version bump
- `archive/` — shipped, wontfix, or duplicate

## Conventions

See [`~/.chronoforge/CONVENTIONS.md`](file:///C:/Users/gezmo/.chronoforge/CONVENTIONS.md) for frontmatter schema, ID format, and lifecycle rules.

## Commands

All Chronoforge feedback slash commands are user-scoped (`~/.claude/commands/feedback-*.md`) and available from any Claude Code session, regardless of which repo you're in. See [`~/.chronoforge/CONVENTIONS.md`](file:///C:/Users/gezmo/.chronoforge/CONVENTIONS.md) for the full command reference.
